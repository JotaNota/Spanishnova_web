# Roadmap

Status: Draft
Date updated: 2026-05-27

## Purpose

Show the route from current state to launch and growth.

GitHub Issues are the execution units.

## Phase 0 - Setup

Goal: make the project ready for agent-based development.

Status: In progress

User stories:

- Create documentation scaffold.
- Configure agent contract.
- Create agent prompts.
- Resolve content roadmap format.
- Create backlog, roadmap, and current state docs.

Acceptance gate:

- `AGENTS.md` exists.
- Agent prompts exist.
- Core docs exist.
- Current state is documented.
- GitHub Issues workflow is defined.
- Git branch and PR workflow is defined.
- GitHub PR template exists.
- Branch protection for `main` is planned.

## Phase 1 - Discovery

Goal: confirm the real WordPress site state.

Status: Not started

User stories:

- Confirm active theme.
- Confirm active plugins.
- Inventory content counts by CPT.
- Inventory taxonomy terms.
- Inventory menus and key pages.
- Confirm analytics and SEO plugin state.

Acceptance gate:

- `docs/current-state.md` reflects verified WordPress state.
- GitHub Issues exist for gaps.

## Phase 2 - Content Architecture and SEO

Goal: define the publishing system.

Status: Not started

User stories:

- Define internal linking rules.
- Define SEO metadata rules.
- Define schema requirements.
- Define content brief format.
- Define publishing cadence.
- Define editorial QA checklist.

Acceptance gate:

- Content rules are documented.
- Content production agents have valid templates and roadmaps.
- GitHub Issues exist for implementation work.

## Phase 3 - Theme Build

Goal: improve the custom theme for structured learning content.

Status: Not started

User stories:

- Audit CPT archive templates.
- Audit CPT single templates.
- Improve reading experience.
- Improve taxonomy pages.
- Improve search experience.
- Add newsletter/signup placement.
- Add internal linking components.

Acceptance gate:

- Theme changes pass QA.
- SEO and accessibility checks pass.
- Current state is updated.

## Phase 4 - Content Production

Goal: produce structured content at a sustainable pace.

Status: Not started

User stories:

- Generate grammar drafts.
- Generate vocabulary drafts.
- Generate reading drafts.
- Generate conversation drafts.
- Create practice agent.
- Create resources agent.
- Review and approve draft batches.

Acceptance gate:

- Drafts exist under `docs/content-system/content/`.
- Editorial QA is complete.
- Publishing candidates are selected.

## Phase 5 - Launch Readiness

Goal: prepare the site for public launch.

Status: Not started

User stories:

## Phase 6 - Growth

Goal: improve SEO, retention, and monetization readiness.

Status: Not started

User stories:

- Track indexed pages.
- Track SEO traffic.
- Track content velocity.
- Track retention.
- Track internal linking coverage.
- Track newsletter/signups.
- Define monetization readiness.

Acceptance gate:

- Metrics are reviewed on a set cadence.
- Backlog is reprioritized from evidence.
