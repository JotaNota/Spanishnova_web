# Delivery Phases

Status: Draft scaffold

## Phase 0 - Setup

- Repository and local WordPress theme available.
- Documentation scaffold created.
- Existing content system preserved.

## Phase 1 - Discovery

- Confirm audience, search intent clusters, content types, conversion paths, integrations, and hosting constraints.
- Audit active WordPress theme, plugins, CPTs, taxonomies, templates, and current site behavior.

## Phase 2 - Content Architecture and UX

- Define information architecture, template rules, internal linking model, editorial fields, and content lifecycle.
- Define SEO plan, content roadmap, and measurement approach.

## Phase 3 - Build

- Implement approved theme changes.
- Extend content templates, archives, single views, navigation, and editorial support.
- Add checks where practical.

## Phase 4 - QA and Launch

- Cross-browser and device QA.
- Accessibility and performance checks.
- SEO checks for indexability, metadata, schema, sitemaps, and internal links.
- Backup, rollback, DNS/cache, and analytics verification.

## Phase 5 - Growth and Maintenance

- SEO and content iteration.
- Internal linking expansion.
- Editorial velocity tracking.
- Theme and plugin update cadence.
- Monetization readiness work.
