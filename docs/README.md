# SpanishNova Documentation Index

This folder documents SpanishNova as a WordPress site with structured editorial architecture and a custom theme.

## Core documents

- `project-charter.md` - purpose, roles, goals, metrics, constraints.
- `requirements.md` - functional and non-functional requirements.
- `phases.md` - delivery phases and acceptance gates.
- `backlog.md` - planning view backed by GitHub Issues.
- `roadmap.md` - phased route from setup to growth.
- `current-state.md` - what exists, what is in progress, and what remains.
- `architecture.md` - WordPress site architecture, content model, theme boundary.
- `agents.md` - agent roles, workflow, and development coordination.
- `workflows/wordpress.md` - WordPress development and editorial workflow.
- `workflows/git.md` - repository workflow.
- `decisions/` - architecture decision records.

## Discipline folders

- `content/` - content strategy, editorial model, page briefs, content operations.
- `content-system/` - existing agent-assisted content system, templates, roadmaps, and source content.
- `agent-system/` - individual development agent prompts.
- `inventory/` - current site inventory and audits.
- `ux-ui/` - UX direction, templates, accessibility, responsive notes.
- `marketing/` - SEO, analytics, conversion tracking, newsletter/signups.
- `devops/` - hosting, backups, deployment, observability.
- `checklists/` - QA, launch, security, and maintenance checks.

## Documentation rules

- Prefer short living documents over large stale specs.
- Record dates for decisions and launch assumptions.
- Mark unknowns as `TBD`.
- Keep `docs/content-system/` intact unless a task targets it.
